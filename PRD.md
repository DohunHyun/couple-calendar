# PRD — Couple Calendar

> 제품 요구사항 문서. **기획 의도(target)가 정본**이며, 각 항목의 **현재 구현 상태**를 함께 표기한다.
> 이전 버전(v1.0)은 "현재 코드가 하는 것"을 의도처럼 기록한 오류가 있었다. v2.0은 **나현 기획서 + 12대 명세를 정본으로** 두고, 실제 코드 정독으로 확인한 상태를 병기한다.
> 기술 상세는 [spec.md](spec.md) 참고.

- **문서 버전**: 2.0 (intent-based)
- **최종 수정**: 2026-06-27
- **상태 범례**: ✅ 충족 / ⚠️ 부분 / ❌ 미충족 / 🐛 버그

---

## 1. 제품 개요

연인 두 사람이 하나의 그룹으로 묶여 **일정을 함께 보면서도, 각자의 사생활은 지키는** 모바일 웹 캘린더.

핵심 가치 세 가지:
1. **공유** — 둘이 함께하는 일정을 한 화면에서.
2. **사생활 보호** — 개인 일정은 작성자 본인에게만.
3. **관계 맥락** — 연애 시작일 기반 D-Day를 곁에.

사용자: 기획자 **나현(Galaxy S26)**, 개발자 **도훈(iPhone)** 같은 연애 중 커플. 모바일 웹 우선, 라이트 모드 미니멀 UI.

## 2. 문제 정의

일반 캘린더 공유는 **전부 공개**(사생활 없음) 또는 **완전 분리**(공유 일정 중복 입력) 양극단으로 흐른다. 커플에게 필요한 건 **"공유는 같이, 개인은 따로"가 한 캘린더 안에서 자연스럽게 나뉘는 것**. 본 제품은 일정마다 `SHARED / PRIVATE` 카테고리로 이를 해결한다.

## 4. 페르소나 & 핵심 시나리오

1. 도훈이 카카오/구글로 로그인 → **초대코드 생성** → 나현에게 공유.
2. 나현이 로그인 → **초대코드 입력** → 두 사람 자동 연결(SSE 실시간 전환).
3. 각자 닉네임·연애 시작일 설정(건너뛰기 가능).
4. 메인 달력에서 날짜 탭 → 하단 시트로 그 날 일정.
5. "주말 데이트"는 **SHARED** → 둘 다 보임. "병원 예약"은 **PRIVATE** → 본인만.

## 5. 사용자 플로우 (온보딩 단계)

서버 `onboardingStage`로 화면 분기: `로그인 → LINK → PROFILE → MAIN`.

- `LINK`: 커플 미연결. **연동 선택 화면**(생성 / 입력).
- `PROFILE`: 연결됨, 프로필 미완료.
- `MAIN`: 메인 달력.

화면 단계: 로그인(UI-ON-001) → 연동선택(UI-ON-002) → 코드생성(UI-ON-003, 소켓대기) / 코드입력(UI-ON-004, 검증) → 프로필(UI-ON-005).

## 6. 핵심 제품 규칙 (불변 비즈니스 룰)

구현 매핑은 [spec.md §5](spec.md).

- **R1. 1인 1커플** — 이미 커플인 사용자는 다른 코드로 연결 불가.
- **R2. 자가 연결 불가** — 자기 초대코드로는 join 불가.
- **R3. 연동 상태 분리** — 생성 시 생성자 couple_id는 NULL, join 성공 시 양쪽 동시 매핑.
- **R4. 가시성** — 일정/카테고리는 *본인 소유* 또는 *같은 커플의 SHARED* 만 보인다.
- **R5. 수정/삭제 권한** — 본인 소유 또는 같은 커플의 SHARED 일정만.
- **R6. 공유 삭제 확인** — 남이 등록한 SHARED 일정 삭제 시 "[작성자]님이 등록한 공유 일정입니다. 함께 삭제하시겠습니까?" 확인.
- **R7. 구글 비파괴** — 구글 일정 삭제는 `is_hidden` 마스킹, 구글 API DELETE 금지.
- **R8. 닉네임 ≤10자**.
- **R9. 색상 12색 파스텔 제한**.
- **R10. 메인 달력 순백 배경 유지**.
- **R11. 제목 ≤50자 / 메모 ≤200자**.
- **R12. 푸시 분기** — PRIVATE→본인, SHARED→커플 2인.

## 8. 비범위 (Out of Scope)

- **3명 이상 그룹** — 1인 1커플 고정.
- **채팅/메신저**.
- **구글 외 외부 캘린더**(Apple/Outlook).
- **구글로의 쓰기(양방향 동기화)** — 읽기 전용.
- **가입 없이 둘러보기(프리뷰)** — 의도상 제외(현재 G5로 잔존).

## 9. 향후 로드맵 / 열린 질문

- G1~G5 처리(위 우선).
- 커플 해제(연결 끊기) 플로우 — `leaveCouple()`만 존재, 정책/UI 미정.
- 일정 반복(매주/매월), 기념일(100일·1주년) 자동 알림.
- 성공지표 계측(연동 완료율, 주간 활성 커플, 커플당 공유 일정 수) — 현재 측정 없음.
